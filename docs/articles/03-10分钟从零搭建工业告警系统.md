# 10 分钟从零搭建一个工业告警系统——EdgeLiteGateway 规则引擎实战

> 本文通过一个完整的实战案例，演示如何用 EdgeLiteGateway 从设备接入到告警通知全链路打通。不需要写一行后端代码，纯配置驱动。适合运维工程师、项目经理和想快速验证 IoT 方案的开发者。

## 场景描述

某工厂有一台 Modbus TCP 温控设备，需求：
1. 实时监控温度、湿度、压力三个测点
2. 温度超过 80°C 持续 30 秒 → 发钉钉告警
3. 温度超限且湿度低于 30% → 发企业微信告警（火灾风险）
4. 所有数据写入 InfluxDB，前端实时展示

## Step 1：Docker 一键启动

```bash
git clone https://github.com/suoten/EdgeLiteGateway.git
cd EdgeLiteGateway
cd web && npm install && npm run build && cd ..
cd docker && docker compose up -d
```

等待约 1 分钟，打开 http://localhost:3000，用 `admin/admin123` 登录。

## Step 2：创建设备

进入 **设备管理 → 新建设备**：

| 字段 | 值 |
|------|---|
| 设备名称 | 温控设备-01 |
| 协议 | modbus_tcp |
| IP 地址 | 192.168.1.100 |
| 端口 | 502 |
| 从站 ID | 1 |
| 采集间隔 | 5 秒 |

添加三个测点：

| 测点名 | 寄存器地址 | 数据类型 | 读写 |
|--------|----------|---------|------|
| temperature | 40001 (Holding Register) | FLOAT32 | 只读 |
| humidity | 40003 | FLOAT32 | 只读 |
| pressure | 40005 | FLOAT32 | 只读 |

点击保存，设备自动开始采集。5 秒后设备状态变为"在线"，测点值实时刷新。

## Step 3：创建规则

### 规则 1：温度超限持续告警

进入 **规则管理 → 新建规则**：

| 字段 | 值 |
|------|---|
| 规则名称 | 温度超限告警 |
| 严重级别 | 严重 |
| 设备 | 温控设备-01 |
| 逻辑 | AND（多条件时） |

添加条件：

| 测点 | 运算符 | 阈值 |
|------|--------|------|
| temperature | > | 80 |

高级设置：

| 字段 | 值 | 说明 |
|------|---|------|
| 持续时间 | 30 秒 | 避免瞬时毛刺误报 |
| 告警收敛 | 开启 | 同一故障不重复告警 |
| 自动恢复 | 开启 | 温度降回正常自动关闭告警 |

通知渠道：选择"钉钉"，填入 Webhook URL 和加签密钥。

### 规则 2：复合条件告警（脚本模式）

当需要更复杂的逻辑时，切换到脚本模式：

```python
# 温度超限且湿度低 = 火灾风险
result = (
    point_values.get("temperature", 0) > 70
    and point_values.get("humidity", 100) < 30
)
```

脚本运行在 **RestrictedPython 沙箱**中，只能访问 `point_values` 和白名单函数（abs/min/max/round），不能调用 `os.system()` 或 `open()`，安全无忧。

通知渠道：选择"企业微信"。

## Step 4：验证告警

### 模拟触发

如果没有真实设备，用内置模拟器：

创建一个模拟器设备，模式选择 `sine`（正弦波），振幅设为 100，频率设为 0.1。温度值会在 -100 到 100 之间波动，自然触发告警。

### 查看告警

1. **告警管理页面**：实时显示所有告警，支持按级别/状态筛选
2. **WebSocket 推送**：前端页面自动刷新，无需手动刷新
3. **钉钉/企微通知**：手机即时收到告警消息

### 告警生命周期

```
触发 → firing（持续30秒确认） → 通知 → 确认 → 恢复 → recovered
```

如果温度在 30 秒内降回正常，告警不会触发。这就是**持续时间窗口**的价值——过滤掉瞬时毛刺。

## Step 5：数据查询与可视化

### 实时数据

设备详情页自动展示实时测点值，每 5 秒通过 WebSocket 更新。

### 历史趋势

进入 **数据查询**，选择设备和时间范围，ECharts 图表自动渲染。支持：
- 多测点叠加对比
- 统计信息（最大/最小/平均/标准差）
- CSV 导出

### Grafana 集成（可选）

在 **服务管理** 中启用 Grafana，系统自动创建 InfluxDB 数据源。直接在 Grafana 中做更专业的可视化。

## 背后的技术细节

### 数据流全链路

```
Modbus PLC
    │
    ▼ (pymodbus, 5秒轮询)
CollectScheduler
    │
    ├─→ Preprocessor (死区过滤: 变化<0.1°C不上报)
    │
    ├─→ EventBus.publish(PointUpdateEvent)
    │       │
    │       ├─→ InfluxDB批量写入 (1000条/批, 5秒刷新)
    │       │       └─→ 失败? → SQLite缓存队列 (10万条上限)
    │       │
    │       ├─→ WebSocket频道广播 → 前端实时刷新
    │       │
    │       ├─→ RuleEvaluator
    │       │       ├─→ 规则缓存 (5秒TTL)
    │       │       ├─→ Cython加速条件比较
    │       │       ├─→ 持续时间窗口追踪
    │       │       └─→ 告警收敛 (同规则+设备不重复)
    │       │               │
    │       │               ├─→ AlarmEvent → EventBus
    │       │               │       ├─→ 钉钉通知 (加签验证)
    │       │               │       ├─→ 企业微信通知
    │       │               │       ├─→ 邮件通知 (SMTP)
    │       │               │       └─→ 自定义Webhook
    │       │               │
    │       │               └─→ 自动恢复检测
    │       │
    │       └─→ MQTT转发 (aiomqtt, QoS 1)
    │
    └─→ DeviceStatusEvent (设备上下线)
```

### 为什么告警不会漏？

1. **EventBus 保证投递**：每个消费者有独立队列，慢消费者不丢数据
2. **告警收敛不丢信息**：同一故障只创建一条告警，但记录触发次数和最新值
3. **通知失败重试**：钉钉/企微 API 调用失败会记录日志，不会阻塞事件循环
4. **断网缓存**：InfluxDB 不可用时数据缓存在 SQLite，网络恢复后自动回传

### 为什么告警不会误报？

1. **持续时间窗口**：必须连续满足条件 N 秒才触发
2. **数据质量标记**：采集超时的数据标记为 `quality="timeout"`，可配置是否参与规则评估
3. **死区过滤**：0.01°C 的波动不会触发规则
4. **脚本沙箱**：自定义脚本在 RestrictedPython 中运行，不会因脚本错误导致系统崩溃

## 进阶：表达式计算

如果需要计算派生测点（如温升速率），使用表达式引擎：

```
rate = (temperature - prev_temperature) / collect_interval
```

或者计算露点温度：

```
dew_point = temperature - (100 - humidity) / 5
```

表达式引擎支持：
- 算术运算：`+ - * / % **`
- 数学函数：`abs round min max sqrt ceil floor log`
- 变量引用：`${device_id.point_name}`
- 安全沙箱：只允许白名单函数

## 进阶：数据预处理

在 **数据预处理** 页面，可以为每个测点配置：

| 预处理类型 | 说明 | 典型配置 |
|-----------|------|---------|
| 死区滤波 | 值变化小于阈值时不上报 | 绝对死区: 0.5°C |
| 百分比死区 | 值变化百分比小于阈值时不上报 | 百分比死区: 1% |
| 滑动平均 | N 个采样值的移动平均 | 窗口大小: 5 |
| 中值滤波 | N 个采样值取中值（去毛刺） | 窗口大小: 3 |
| 时间聚合 | 时间窗口内聚合 | 60秒窗口, 取平均值 |

**实战建议**：温度类测点用死区滤波 + 滑动平均，压力类测点用中值滤波去毛刺，流量类测点用时间聚合降采样。
