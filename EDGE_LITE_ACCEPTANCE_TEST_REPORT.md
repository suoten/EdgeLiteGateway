## 三、Bug 清单

### P0 — 致命（阻断性）

| Bug ID | 描述 | 复现步骤 | 影响 | 状态 |
|--------|------|----------|------|------|
| BUG-P0-001 | `edgelite.models.north` 模块缺失 | `from edgelite.api.platforms import router` | 平台集成模块完全不可用，`/api/v1/platforms/*` 全部 500 | 🔴 未修复 |

**复现步骤**:
```bash
python -c "from edgelite.api.platforms import router"
# ModuleNotFoundError: No module named 'edgelite.models.north'
```

**根因分析**: `src/edgelite/services/platform_service.py` 第 10 行导入 `from edgelite.models.north import (Mqtt5Properties, MqttConnectionConfig, ...)` 但 `edgelite/models/north.py` 文件不存在。这些类在整个代码库中未定义。

### P1 — 严重

| Bug ID | 描述 | 复现步骤 | 影响 | 状态 |
|--------|------|----------|------|------|
| BUG-P1-001 | 分页参数 `size` 验证上限过大 | GET `/api/v1/devices?page=1&size=5000` | 单次查询可返回 5000 条记录，可能导致 OOM | 🟡 已知风险 |
| BUG-P1-002 | DELETE 设备接口返回值解包 | `success, error = await svc.delete_device(device_id)` | 如果服务返回非元组类型，触发 500 | 🟡 需防御性编程 |

1. **立即修复 P0 Bug**: 创建 `edgelite/models/north.py` 模块或修改 `platform_service.py` 的导入路径

3. **增强测试覆盖**: 在真实 DB 环境中运行完整测试套件
